# 稼働ゼロなのに毎月課金される“亡霊コスト”

サーバーは全部止めたのに、毎月お金が請求される——消し忘れた固定費（“亡霊”）を見つける**見張り番**のサンプルです。

## このサンプルが示すこと（動画③のしくみ）

1. **金額の大小でなく“増え方”を見る** … 亡霊は少額なので「○○円超えたら警報」では捕まらない。
2. **“先月の同じ日”と比べる** … 今月の積み上がりが、先月の同じ日より急に増えていないか。
3. **“新顔”を見つける** … 先月ゼロなのに今月から課金が始まった項目＝新しい亡霊。
4. **勝手に消さない** … 見つけたら「これ消していい？」と聞く → 人が承認 → そこで消す。

## 動かす

```bash
python watch_cost.py                      # ダミーの請求データで今日を 2026-06-10 として点検
python watch_cost.py --today 2026-06-10   # 日付を指定
```

請求データ `sample_billing.jsonl` は**全部ダミー**です。実際にはここをクラウドの請求APIに差し替えます。

## Claude Code に頼むなら

> `watch_cost.py` を読んで、しきい値（急増とみなす倍率）を 1.5 倍に変えて、検出結果がどう変わるか試して。

## 出力イメージ

```
🐶 見張り番レポート（2026-06-10 / 先月同日と比較）
  ⚠ 急増   バックアップ        先月 $0.06 → 今月 $1.50（25.0倍）
  🆕 新顔   見張り番            先月 $0.00 → 今月 $0.90（先月は無かった課金）
  ✅ 異常なし 固定IP / 保管料

これ、消していい？（承認したものだけ消します）
```
