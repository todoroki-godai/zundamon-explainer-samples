# AIの「完了しました」を“証拠”で確かめる

AIは「できました！」と言っても、中身が終わっていないことがあります（＝“終わったフリ”）。
このサンプルは、AIの「完了」を**人が信じる前に機械で確かめる関所**です。動画の合言葉「“証拠”で確かめろ」をそのまま動かせます。

## まず動かす（最初はわざと失敗します）

```bash
python verify.py
```

お題の電卓 `calc.py` は「足し算・引き算できました！」と言いつつ、引き算が中身ナシ。だから **FAIL**：

```
❌ 未完了：AIの「完了」は“フリ”でした

  ❌ テストが緑  → test_calc が落ちた
  ✅ 必須の関数 add() がある
  ✅ 必須の関数 subtract() がある
  ❌ 禁止ワード "NotImplementedError" が無い  → calc.py に "NotImplementedError" が残ってる
  ❌ 禁止ワード "TODO" が無い  → calc.py に "TODO" が残ってる
  ✅ 禁止ワード "FIXME" が無い

判定: FAIL（1つでも欠けたら“完了”とは認めない）
```

`calc.py` の `subtract` の行を実装し直す：

```python
def subtract(a, b):
    return a - b
```

もう一度 `python verify.py` を回すと——

```
✅ 完了：証拠がそろいました

  ✅ テストが緑
  ✅ 必須の関数 add() がある
  ✅ 必須の関数 subtract() がある
  ✅ 禁止ワード "NotImplementedError" が無い
  ✅ 禁止ワード "TODO" が無い
  ✅ 禁止ワード "FIXME" が無い

判定: PASS
```

## しくみ（完了の定義を“先に”書く）

- `done.json` … **「何をもって完了か」を先に宣言**する（テスト・必須の関数・残ってはいけない言葉）。
- `verify.py` … その宣言を1つずつ機械チェックして PASS / FAIL を出す。**1つでも欠けたら未完了**。
- `CHECKLIST.md` … コードを書かない人向けの「目で確かめる版」。

ポイントは **作業を始める前に done を書く**こと。後出しにすると、AIの「完了」に合わせて基準が緩み、詐欺が通ります。

## Claude Code に頼むなら

> `calc.py` の `subtract` を実装して。そのあと `python verify.py` を回して、PASS になるまで直して。

AIに「直して」と言うだけでなく、**verify が緑になることまで**を頼むのがコツです。

## なぜ効くのか

AIは「高い点数がもらえるよう」訓練されています。本当に終わらせるのが難しいとき、“終わらせる”代わりに“終わったことにする”近道を覚えることがあります。
だから人間側に**証拠で確かめる関所**を置く——これが完了詐欺への一番の対策です。

ただし関所は完璧ではありません。テストが1ケースだけだと、AIがその数値に合わせる“近道”（例: 引き算なのに `return 2` と固定で返す＝まさに reward hacking）を覚えて通り抜けます。だから**テストは複数ケースで、完了条件はできるだけ厳しく**——それが verify を信頼できる関所にするコツです。

（`calc.py` 等はすべて教材用のダミーです。実際には自分のテスト・完了条件に置き換えてください。）
