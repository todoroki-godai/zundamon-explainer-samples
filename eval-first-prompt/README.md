# 点数を決めたらAIが自分でプロンプトを直す

プロンプトを良くしたいとき、**先に「点数の付け方（ものさし）」を決める**と、あとは「直して→測って→また直す」をAIが回せる、という話のサンプルです。

ポイントは順番です。**直し方を先に考えない。測り方を先に固める。**

## 中身

| ファイル | 役割 |
|----------|------|
| `cases.jsonl` | お題（短いメッセージ）と、正解ラベル |
| `prompt.txt` | いま改善したい“分類プロンプト”（最初はわざと雑） |
| `score.py` | **ものさし**。予測と正解を突き合わせて正答率を出す（LLMは呼ばない） |
| `outputs/predictions.sample.jsonl` | 初見でもすぐ動く**お試し予測**（わざと雑）。`predictions.jsonl` を作るとそちらが優先される |
| `outputs/predictions.jsonl` | プロンプトで分類した結果を置く場所（自分で作る） |

## まず動かす（セットアップ不要）

```bash
python score.py
```

最初は**お試し予測**を採点して `6/8` と「外したケース」が出ます（＝ものさしが動く様子）。
ここから下の手順で、自分の予測に差し替えて点数を上げていきます。

## 回し方（Claude Code 前提）

1. **分類させる**：Claude Code に頼む
   > `prompt.txt` のルールで `cases.jsonl` の各 `text` を分類して、`outputs/predictions.jsonl` に `{"id":.., "label":".."}` 形式で出して。
2. **測る**：
   ```bash
   python score.py
   ```
   正答率と「外したケース」が出ます。
3. **直す**：Claude Code に頼む
   > `score.py` が外したケースを見て、`prompt.txt` を直して。
4. **2〜3 を繰り返す**。点数が上がるまでAIが回せます。

## なぜ「ものさし先決め」なのか

雑なプロンプトを“感覚で”直すと、直したつもりで別のケースが壊れる。`score.py` という**動かない事実（点数）**があると、改善が当てずっぽうでなくなる。改善のループ自体はAIに任せられる。
