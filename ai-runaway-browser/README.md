# AIに任せたらブラウザを40個開いて暴走

ゲームの動作チェックをAIに自動で任せたら、裏で“画面のないブラウザ”が15分で40個に増殖してPCが悲鳴を上げた——その**暴れない任せ方**のサンプルです。

## まず「悪い例」と「直した例」を動かす

依存ゼロのシミュレーションです（本物のブラウザは起動しません）。

```bash
python unsafe.py   # 丸投げ版：閉じる指示が無く、ブラウザが40個まで積み上がる
python safe.py     # 安全版：作法＋番犬＋自分の分だけ片付け → 数個で安定
```

## 安全な任せ方の3つ（`safe.py` に実装）

1. **省エネの“作法”を一言わたす** … いちばん軽い設定で開く（本物の起動引数は `safe.py` 末尾のメモ参照）。
2. **何個開いてるか見張る“番犬”** … 同時に開いている数を数え、しきい値（例: 8個）を超えたら止める。
3. **片付けは“自分の分だけ”** … 全部いっぺんに消すと、隣で動いている別の作業まで巻き添えにする。自分が開いた分だけ閉じる。

## Claude Code に頼むなら

> `safe.py` の `MAX_CONCURRENT` を 4 に下げて、番犬が早く止まることを確認して。
> そのあと、本物の Playwright 版（末尾メモ）に書き換えて。

## 教訓

「AIに任せる」は丸投げではない。**暴れない作法を先に渡して、ちゃんと見張る**。便利な道具ほど、走らせる前に道を用意してあげる。
