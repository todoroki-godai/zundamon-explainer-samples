# テストが本物のLLMを呼んでいた話

「テストのつもりが、毎回**本物のLLM（とお金）**を叩いていた」事故を防ぐ仕組みです。

この回のサンプルは、すでに公開している2つのOSSが**そのまま実物**なので、作り直さず本物にリンクします。

## 2層で止める

| 層 | repo | 何をする |
|----|------|---------|
| コミット時（静的チェック・全言語） | **[no-llm-in-tests](https://github.com/todoroki-godai/no-llm-in-tests)** | pre-commit hook。テストコードの中の LLM SDK 呼び出し（`anthropic` / `openai` など）を見つけたらコミットを止める |
| 実行時（ランタイムガード・Python） | **[pytest-no-llm](https://github.com/todoroki-godai/pytest-no-llm)** | pytest プラグイン。テスト実行中に実ネットワーク越しの LLM 呼び出しが起きたら落とす |

## ポイント

- 「うっかり」を**人の注意力で防がない**。コミット時と実行時の2か所で機械的に止める。
- 全部を mock 必須にするのではなく、「本物を**叩いた瞬間だけ**」fail にするのがコツ（テストは書きやすいまま、事故だけ止める）。

導入手順は各 repo の README を見てください。
